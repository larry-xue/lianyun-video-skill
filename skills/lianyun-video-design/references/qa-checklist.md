# 渲染后 QA · 逐 slide 抽帧自检

**完播率的敌人不是内容差,是某一帧观众跟不上 / 觉得糙。** 只看一帧不够,要抽连续帧。

## 抽帧方法

```bash
# 每 ~2s 抽一帧,快速过一遍
for f in 0 60 120 180 240 300; do npx remotion still my-video out/f_$f.png --frame=$f; done
```

对**每一张 slide** 至少抽一个连续帧窗口(优先:slide 起始后 2–4s、长段正文出现处、动画切换密集处)。

## 逐帧检查项

- [ ] **正文不溢出**内容安全区(不低于 y=900,不进右 340px 互动区)。
- [ ] **SVG label 不压数据曲线 / 线段**。
- [ ] **相邻帧换行不漂移**——字体没突然变宽 / 变窄,正文换行位置稳定(动画中尤其查)。
- [ ] **字幕 / 元素不抖动**、不压到下一行。
- [ ] **首帧(frame 0)干净**、完整可见,能直接做分享封面。
- [ ] **数字等宽对齐**(`tabular-nums`)。
- [ ] **留白合理**:一屏正文纵向占 body ≥ 80%,不显空。

## 提交前 checklist

- [ ] `npx tsc --noEmit` 类型通过。
- [ ] 上面的逐 slide 抽帧过了一遍。
- [ ] 首帧可做封面。
- [ ] 脚本已过「去 AI 腔」(`humanizer-zh` 或手动对照其规则)。
- [ ] 视觉如果想再狠一点,过一遍 `impeccable`。

> 已知易回归点:文字换行抖动常出现在「渲染参数 / 字体改动之后」——改完必复查那几帧。
