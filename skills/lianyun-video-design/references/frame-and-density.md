# Frame chrome、安全区、密度硬标准

以「炼云讲堂」的 1920×1080 横屏为例。你的 `design.md` 可以改尺寸,但这套「安全区 + 密度」的思路通用。

## 画布与安全区(1920×1080)

```
1920px
┌──────────────────────────────────────────┬──────────────┐
│ 01 / HOOK                    ┌────────┐  │              │
│  ← section eyebrow(mono)     │ @brand │  │  平台互动区   │
│                              └────────┘  │  ~340px      │
│   ◆ 内容安全区 1508 × 760                 │  点赞/评论/   │
│   top:140  bottom:180  left:72           │  分享覆盖     │
│                              右边界 x=1580 │  禁放可读内容 │
│ FOOTER                       01 / 08      │              │
└──────────────────────────────────────────┴──────────────┘
```

- **top 140**:给 section eyebrow + brand tag 的 chrome 区。
- **bottom 180**:内容**绝对**不低于 y=900(横屏时底部易被 UI 挡)。
- **left 72**:内容左起点。全宽色带用负边距 `left:-72` 突出,body `overflow: visible`。
- **默认:左右对称满宽**(left/right 各 72,内容宽 1776),视觉均衡,适合大多数平台和独立视频。**别默认留右侧 340px**——那会让画面左重右空,看着像宽度出了 bug。
- **抖音横屏(可选)**:抖音横屏观看时右侧点赞/评论/分享 UI 会盖住约 340px。**只有专门发抖音横屏**才给 `SlideFrame` 传 `douyinSafeZone`,内容缩到 1508 宽,把正文/数据/关键图挡在安全线内(brand tag / 页码一起内收)。上面那张 ascii 图画的就是 `douyinSafeZone=true` 的布局。

## 背景

整片统一背景(例:米白纸 `#F5F2EB` + 轻噪点纹理),封面和内容 slide 同一套参数,视觉连续。这是系列辨识度的地基。

## 密度硬标准

- **字号**:主标题 56–96px、secondary 26–38px、card padding 28–36px。
- **纵向占比**:一屏正文纵向占 body(约 760px)**≥ 80%**,底部留白 **≤ 150px**——留白太多显空。
- **数字对齐**:金额 / 百分比 / 倍数用 `fontVariantNumeric: "tabular-nums"` 等宽对齐。
- **图表 label**:SVG 里的 label **不准穿过数据曲线 / 线段**——要么移到 SVG 外用 HTML 白底徽章,要么放到曲线远离的侧边。

## 中英混排

- 保留英文技术术语(`Agent` / `token` / `API`),不硬翻。
- 不翻译会改变意思的数字单位(`200K` 不写成「二十万」如果原语境是 K)。
- 默认正文用中文,术语留英文。
